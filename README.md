## Реализованная функциональность

### Механизм автоматического создания задач

Добавлена поддержка инструкций (instructions) для автоматического пересоздания задач. 
Связь между задачей и инструкцией — "один к одному".

**Создание инструкции:**
- При создании новой задачи
- Отдельно с последующей привязкой к существующей задаче

**Как это работает:**
При создании инструкции сервис автоматически:
1. Создает новую задачу на будущую дату согласно правилам инструкции
2. Привязывает созданную задачу к инструкции

**Планировщик (Scheduler):**
Для обеспечения бесконечного цикла создания задач и предотвращения перегрузки БД, реализован фоновый процесс. Ежедневно в 10:00 он:
1. Проверяет наличие задач, запланированных на текущий день с привязанными инструкциями
2. Создает новые задачи согласно инструкциям
3. Перепривязывает инструкции к вновь созданным задачам

**Дополнительно:**
1. Реализована атомарность при создании нескольких сущностей при работе одной ручки
2. Добавлена валидация на количество вводимых символов

### Изменения в коде

#### 1. Новая сущность: Инструкция (Instruction)

Инструкция хранит правило, по которому задача должна быть пересоздана.

**Связь с задачей:** Один к одному (One-to-One)

#### 2. Новое поле в задаче: Deadline

В сущность `Task` добавлено обязательное поле `deadline`, которое используется для определения срока действия задачи и
расчёта даты следующего запуска планировщика.

#### 3. API для работы с инструкциями

Обновленные ручки для работы с инструкциями представлены в Swagger.

При реализации ручек для сущности `Instruction` были созданы соответствующие репозитории для взаимодействия с БД, 
usecase'ы и hardler'ы для обработки запросов

#### 4. Планировщик
- Запускается в отдельной горутине при старте приложения
- Использует библиотеку `github.com/robfig/cron/v3`
- Graceful shutdown — остановка планировщика при завершении работы приложения

#### 5. Транзакционность:
- Создание задачи вместе с инструкцией выполняется в одной БД-транзакции
- При ошибке на любом этапе все изменения откатываются
- Работа планировщика также обёрнута в транзакции для каждой инструкции

#### 6. Миграции:
- Добавлена миграция `0002_create_instructions.up.sql` для создания таблицы `instructions`
- Обновлена миграция `0001_create_tasks.up.sql` — добавлено поле `deadline`

# Task Service

Сервис для управления задачами с HTTP API на Go.

## Требования

- Go `1.23+`
- Docker и Docker Compose

## Быстрый запуск через Docker Compose

```bash
docker compose up --build
```

После запуска сервис будет доступен по адресу `http://localhost:8080`.

Если `postgres` уже запускался ранее со старой схемой, пересоздай volume:

```bash
docker compose down -v
docker compose up --build
```

Причина в том, что SQL-файл из `migrations/0001_create_tasks.up.sql` монтируется в `docker-entrypoint-initdb.d` и применяется только при инициализации пустого data volume.

## Swagger

Swagger UI:

```text
http://localhost:8080/swagger/
```

OpenAPI JSON:

```text
http://localhost:8080/swagger/openapi.json
```

## API

Базовый префикс API:

```text
/api/v1
```

Основные маршруты:

- `POST /api/v1/tasks`
- `GET /api/v1/tasks`
- `GET /api/v1/tasks/{id}`
- `PUT /api/v1/tasks/{id}`
- `DELETE /api/v1/tasks/{id}`
